基础使用

创建 MediaPlayer

可以用 new 和默认 create 方式构造

设置媒体源

根据媒体来源不通过，可以通过几种不同的方法记载资源

注意对于 9.0 版本对于http请求有访问限制，只能使用 https ，否则将抛出java.io.IOException: Cleartext HTTP traffic to demo.com not permitted 异常，如果需要支持 http 则在manifest.xml中设置 android:usesCleartextTraffic="true"

基本操作

状态切换

  • Idle 在新MediaPlayer新建或者reset操作发生时处于该状态，需要注意new对象一开始产生时由于播放引擎还未启动，所以该状态下发生的error并不会产生异常回调
  • Initialize 设置资源资源后状态，该状态下可以现在异步 prepareAsync 和 prepare 将MeidaPlayer状态转为 Prepared
  • Prepared 媒体资源就绪状态，该状态下可以对播放引擎的属性进行操作，并且可以对播放操作进行变更比如 seekTo 、stop、stop
  • Started 播放状态，该状态可以进行播放操作，当播放完成时，取决于是否进入循环模式，如果是则在完成后自动再进入 Started 重新播放，如果不是则进入 PlaybackCompleted
  • Paused 暂停播放状态，也可以对播放操作行为进行操作
  • Stop 播放终止状态，释放播放状态，可以重新进入Prepared后再次进入Started状态。
  • PlaybackCompleted 播放完毕状态，处于待处理状态，可以对资源终止播放或者重新播放
  • End MediaPlayer 播放引擎资源释放，该状态后不能进行任何状态转化
  • Error 播放引擎异常状态，可以reset 重新进入状态
  • Preparing 介于 Prepared 转换前的短暂状态，在进行异步执行过程中，会有短暂延时进入 Prepared，完成时通过 OnPreparedListener.onPrepared 进行回调

播放 start

执行后，将进入Started 状态，可以通过isPlaying 进行判断。

在 Prepared、Pause、PlaybackComplete 状态时，可以调用进入

循环播放

播放完成后不会进行OnCompletionListener 回调，直接进入重新播放，否则false 将进入 PlaybackCompleted 状态

暂停 pause

暂停播放，在执行前可以先判断 mediaPlay.isPlaying() 是否处于 Started 状态，执行成功后将处于 Paused 状态

停止 stop

对 Started、Pause、PlaybackComplete 状态终止，转为 Stopped 状态，该状态下只能重新进入 Prepared 后才能进行播放

重置 reset

重置播放资源，reset 后将进入idle状态，需要重新setDataSource

进度调整 seekTo

调整到指定时间戳播放位置，seekTo 不会改变当前的状态

释放 release

如果不需要 MediaPlayer 则一定要调用该方法进行播放引擎资源的释放。执行后将进入End状态并不可恢复。

监听器

根据状态MediaPlayer 提供了几种监听器

  • OnPreparedListener 当进入 Prepared 状态时回调

  • OnErrorListener 当进入 Error 状态时通知

    如果返回值为 true 则表示错误被当前处理，不会进行 OnCompletionListener 回调。

  • OnCompletionListener 播放完成回调，如果处于循环播放模式，则在结束后不进行回调

  • OnBufferingUpdateListener 当处于网络获取源时，该回调反应缓冲进度
