Android中存储目录API
发表于 2016-07-22

Android 中对目前切分了比较多的分类，记忆起来比较乱，记录下烂笔头，防止后面忘记：

getCacheDir

应用内部缓存目录，可以通过应用属性，清除缓存进行清理。
如果不希望被清理软件清理不要放这里了

  • 应用卸载则清除
  • 普通用户不可访问
  • 不需要额外读写权限

getFileDir

应用内部文件目录

  • 应用卸载则清除
  • 普通用户不可访问
  • 不需要额外读写权限

getExternalCacheDir

和 getCacheDir 呼应，只是该缓存存在外存中。

  • 应用卸载则清除
  • 用户可以外存中访问
  • 应用详情中清除缓存``清除数据可被清除
  • 不需要额外读写权限

getExternalCacheDirs

和 getCacheDir 类似，获取所有的cache目录

  • 需要4.4及其以上支持

getExternalFilesDir

和 getFileDir 呼应也是存在外存上

  • 应用卸载则清除
  • 用户可以外存中访问
  • 应用详情中清除数据可被清除
  • 不需要额外读写权限

getDataDir

应用内部数据存储根目录

  • 应用卸载则清除
  • 普通用户不可访问
  • 不需要额外读写权限

getNoBackupFilesDir

对于备份，要求不进行备份的文件目录

  • 应用卸载则清除
  • 用户不可访问
  • 不需要额外读写权限
  • Android5.0及以上API可用

getCodeCacheDir

用于存储一些优化代码，比如odex 之类的

  • 应用卸载则清除
  • 用户不可访问
  • 不需要额外读写权限
  • Android7.0及以上API可用

getPackageCodePath、getPackageResourcePath

目前两个方法都是同一个地址，系统默认的一些应用代码和资源的文件，对于普通开发不需要，base.apk 存放于此

  • 应用卸载则清除
  • 用户不可访问
  • 不需要额外读写权限
  • Android7.0及以上API可用

getDataDirectory

内部存储 /data/ 顶层目录，普通开发不需要

  • 用户不可访问
  • 需要系统级应用权限

getDownloadCacheDirectory

系统级的缓存，放置在内部存储中。普通开发用不到

  • 用户不可访问
  • 需要系统级应用权限

getRootDirectory

顶层系统目录，普通开发用不到

  • 用户不可访问
  • 只有读取权限，无法进行写入

Environment.getExternalStoragePublicDirectory

外存目录

  • 用户可访问

  • 需要添加权限声明，并且需要动态申请权限

  • 应用卸载不会清除

getExternalStorageDirectory

外存根目录

  • 用户可访问
  • 需要添加权限声明，并且需要动态申请权限

外部存储的状态

外部存储除了需要权限，还需要判断挂载状态

分别有几种状态：

  • MEDIA_UNKNOWN：未知存储状态，例如路径没有由已知存储支持时
  • MEDIA_REMOVED：存储媒体被移除
  • MEDIA_UNMOUNTED：存储媒体没有挂载
  • MEDIA_CHECKING：如果媒体存在并正在检查磁盘
  • MEDIA_NOFS：不支持的文件系统
  • MEDIA_MOUNTED：媒体已经挂载，并且可读/写
  • MEDIA_MOUNTED_READ_ONLY：媒体已经挂载，只读
  • MEDIA_SHARED：在通过USB共享
  • MEDIA_BAD_REMOVAL：在没有挂载前存储媒体已经被移除
  • MEDIA_UNMOUNTABLE：存储媒体无法挂载
  • MEDIA_EJECTING：存储媒体处于被弹出的过程
