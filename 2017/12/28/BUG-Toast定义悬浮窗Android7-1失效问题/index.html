发表于 2017-12-28|Android

Toast悬浮窗在 Android 7.1 突破权限失效问题

标签（空格分隔）： 悬浮窗

Toast类型悬浮窗问题

通常为了避免悬浮窗在6.0后申请ALERT权限问题,会使用Toast类型来做弹窗图层(具体原因参照代码 PhoneWindowManager#checkAddPermission方法),但是目前在7.1上,该方式已经失效.具体原因做以下分析.

WindowManager新增参数hideTimeoutMilliseconds

官方解释:

以及 在Toast上对setDuration(int duration)的调整:

TN对象中的调整

也就是说为了保证Toast能正常隐藏,7.1后对Window新增一控制参数,来增强对Toast Window的消失及时性.
而这种控制方式是建立在Window核心代码上,并且对于该超时时间的控制并不提供用户自定义.

PhoneWindowManager增加对Toast类型LayoutParams的hideTimeoutMilliseconds校验

如果仅仅是增加hideTimeoutMilliseconds超时参数的话,是可以进行无限大时间来欺骗超时,但是在对Window进行显示过程中,PhoneWindowManager对该参数值进行校验并修正.

注: 同时Toast Window动画设置是无效的,这里被重置了!

解决方案

对于目前情况,暂时没有很好的突破口,只能建议使用Alert权限申请的方式进行.
欢迎有更好的建议或者漏洞提供.
