BUG-Paging框架PageListAdapter与BaseRecyclerViewAdapterHelper框架BaseQuickAdapter冲突
发表于 2019-11-28|Android

在之前的项目中由于引入了 BaseRecyclerViewAdapterHelper 框架的支持，同时又需要使用 Paging 框架的内容，造成在同一个 RecyclerView.Adapter 使用上冲突。

在 Paging 框架 PageListAdapter 中发现，对于数据的处理和触发都是通过 AsyncPagedListDiffer 进行处理，因此这里尝试做下迁移。

大致调整如下：

牺牲一部分Paging的特性，比如对于maxsize支持无用（由于外部持有data缓存），获得在Paging上懒加载和分页能力的支持。主要调整如下：

  • 实现 ListUpdateCallback 接口，用于在Paging数据源数据变化时同步更新到 BaseQuickAdapter 中
  • 创建 Paging 数据差分工具对象 AsyncPagedListDiffer，AsyncPagedListDiffer 作为构造，在发生数据变化时进行响应。
  • 提供 submitList PageList 数据源注册入口，在订阅到 PageList 后，将其提交给 AsyncPagedListDiffer
  • 重写 BaseQuickAdapter.getItem 方法，当执行该方法时同时触发 AsyncPagedListDiffer.getItem 方法，该方法触发Paging数据预加载。
